# Примеры использования Construction Maps MCP

## 🎯 Сценарии использования для строительства

### 1. 🏗️ Предпроектный анализ участка

**Задача**: Нужно оценить земельный участок для строительства

```
Claude, проанализируй участок 50:21:0010401:14 для строительства:
1. Получи границы и основную информацию
2. Найди ближайшую инфраструктуру (школы, больницы, магазины)
3. Рассчитай площадь участка
4. Создай карту с маркерами инфраструктуры
```

**Используемые tools**:
- `cadastre_get_boundaries` - границы GeoJSON
- `cadastre_get_info` - кадастровая информация
- `infrastructure_find_nearby` - поиск объектов в радиусе
- `geometry_calculate_area` - точная площадь в м²
- `visualization_generate_static_map` - карта для отчета

**Результат**: Полный отчет с границами, площадью, инфраструктурой и картой

---

### 2. 🚚 Логистический анализ

**Задача**: Оценить расстояния от участка до поставщиков

```
У меня участок по адресу "Москва, ул. Строителей, 15".
Рассчитай расстояния до:
- Складов: "Подольск, Промзона Север", "Химки, Складской комплекс"
- Производителей: "Мытищи, ул. Индустриальная, 10"
- Клиента: "Москва, Кутузовский проспект, 30"

Покажи результаты таблицей с расстояниями в км.
```

**Используемые tools**:
- `geocode_address_to_coords` - координаты участка
- `infrastructure_calculate_distances` - матрица расстояний
- `geometry_measure_distance` - точные геодезические расстояния

**Результат**: Таблица с расстояниями для планирования логистики

---

### 3. 📊 Отчет с картами для заказчика

**Задача**: Создать отчет с визуализацией

```
Создай отчет по участку 77:01:0005001:1234:
1. Спутниковый снимок участка
2. Схематическая карта с границами и маркером
3. Информация о площади и назначении
4. Экспорт границ в GeoJSON для ГИС

Карты должны быть пригодны для печати.
```

**Используемые tools**:
- `infrastructure_get_satellite_image` - спутник
- `visualization_generate_static_map` - карта с границами
- `cadastre_get_info` - характеристики
- `visualization_export_geojson` - экспорт для QGIS

**Результат**: Профессиональный отчет с картами и данными

---

### 4. ⚠️ Анализ пересечений участков

**Задача**: Проверить конфликт границ между участками

```
У меня два соседних участка:
- Участок A: 50:21:0010401:10
- Участок B: 50:21:0010401:11

Проверь:
1. Пересекаются ли их границы?
2. Если да - какая площадь пересечения?
3. Создай карту с обоими участками и зоной конфликта
```

**Используемые tools**:
- `cadastre_get_boundaries` (x2) - границы обоих участков
- `geometry_check_intersection` - проверка пересечения
- `geometry_calculate_area` - площадь конфликтной зоны
- `visualization_generate_static_map` - визуализация конфликта

**Результат**: Детальный анализ конфликта границ с картой

---

### 5. 🛡️ Проверка санитарно-защитной зоны (СЗЗ)

**Задача**: Рассчитать буферную зону вокруг объекта

```
На участке 50:30:0020101:50 планируется промышленный объект.
СЗЗ = 500 метров.

1. Создай буферную зону 500м вокруг участка
2. Найди все жилые дома в этой зоне
3. Рассчитай площадь СЗЗ
4. Экспортируй зону в GeoJSON для проектирования
```

**Используемые tools**:
- `cadastre_get_boundaries` - границы участка
- `geometry_buffer` - буфер 500м
- `infrastructure_find_nearby` - поиск жилых домов
- `geometry_calculate_area` - площадь СЗЗ
- `visualization_export_geojson` - экспорт зоны

**Результат**: СЗЗ с анализом потенциальных конфликтов

- "Москва, Тверская, 1"

## 🎓 Советы по использованию

### 1. Используйте кеширование

MCP сервер кеширует все запросы:
- Кадастровые данные: 30 дней
- Геокодирование: 7 дней
- Инфраструктура: 1 день

Повторные запросы выполняются мгновенно!

### 2. Комбинируйте tools

Claude автоматически выстраивает цепочки:
```
"Найди участок по адресу и покажи его на карте"
→ geocode_address_to_coords
→ cadastre_search_by_address
→ cadastre_get_boundaries
→ visualization_generate_static_map
```

### 3. Экспортируйте данные

Все результаты можно сохранить:
- GeoJSON для QGIS/ArcGIS
- JSON для архивов
- Markdown для отчетов
- URL картам для презентаций

### 4. Указывайте единицы

Для расстояний и площадей:
- Расстояния: метры (м) и километры (км)
- Площади: м² и гектары (га)
- Буферы: всегда в метрах

## 📞 Что делать при ошибках?

### "Кадастровый номер не найден"
- Проверьте формат: `50:21:0010401:14` (без пробелов)
- Участок может быть не в НСПД (только публичная кадастровая карта)

### "Адрес не найден"
- Укажите полный адрес: "Город, улица, дом"
- Используйте стандартные сокращения: "ул.", "пр-т", "д."

### "Yandex Geosearch not implemented"
- Поиск инфраструктуры имеет ограничения
- Для production используйте 2GIS или OpenStreetMap API

### Медленная работа
- Первый запрос идёт к API (медленно)
- Повторные запросы из кеша (быстро)
- Проверьте скорость интернета для API Росреестра
