## 1. Градостроительный план земельного участка (ГПЗУ)

### Нормативная база
- **ГрК РФ Статья 57.3** — ГПЗУ действителен 3 года
- **Обязательное содержание**: границы участка, кадастровый номер, минимальные отступы, красные линии, инженерные сети, ограничения использования

### Что даёт MCP
```
1. cadastre_get_boundaries(cadastral_number)
   → Границы участка для расчёта минимальных отступов

2. cadastre_get_info(cadastral_number)
   → Площадь, ВРИ, категория земель, кадастровая стоимость

3. geometry_calculate_area(boundaries)
   → Проверка соответствия кадастровой площади

4. infrastructure_find_nearby(lon, lat, radius=500)
   → Анализ существующей инфраструктуры

5. visualization_export_geojson(cadastral_number)
   → Экспорт для QGIS / AutoCAD Civil 3D
```

### Применение
- Проверка соответствия категории земли планируемому назначению
- Выявление ограничений и обременений (санитарные зоны, охранные зоны)
- Сопоставление кадастровых границ с фактическими данными геодезии

## 2. Расчёт коэффициентов застройки (СП 42.13330.2016)

### Нормативная база
**СП 42.13330.2016** "Градостроительство. Планировка и застройка":
- **КЗ** (коэффициент застройки) = Площадь застройки / Площадь участка
- **КП** (коэффициент плотности) = Площадь всех этажей / Площадь участка

**Нормативы:**
- Многоэтажная жилая застройка: КП ≤ 1,2 (новая), КП ≤ 1,6 (реконструкция)
- Малоэтажная застройка: КП ≤ 0,8
- Частные дома с приусадебными участками: КП ≤ 0,4
- Промышленная застройка: КП ≤ 2,4

### Что даёт MCP
```python
# Пример расчёта для Claude:
1. geometry_calculate_area(site_boundary)
   → Площадь участка = 10000 м²

2. Если ВРИ = "многоэтажная жилая застройка":
   Макс. КП = 1.2
   Макс. общая площадь этажей = 10000 × 1.2 = 12000 м²

3. Проверка проекта:
   geometry_calculate_area(building_footprint)
   → Площадь застройки = 2000 м²

   КЗ = 2000 / 10000 = 0.2 (20% застройки) ✅

4. При этажности 5 этажей:
   Общая площадь этажей = 2000 × 5 = 10000 м²
   КП = 10000 / 10000 = 1.0 ✅ (в пределах нормы 1.2)
```

### Применение
- Автоматическая проверка соответствия проекта нормативам
- Расчёт максимально допустимой застройки
- Оптимизация этажности при заданной площади участка

## 3. Санитарно-защитные зоны (СанПиН 2.2.1/2.1.1.1200-03)

### Нормативная база
**СанПиН 2.2.1/2.1.1.1200-03** "Санитарно-защитные зоны":
- **СЗЗ** — территория с особым режимом использования, отделяющая промышленную площадку от жилой застройки
- Размеры СЗЗ зависят от класса опасности предприятия (от 50 м до 1000 м)

### Сценарий: Размещение производственного объекта

**Задача**: Разместить завод класса III опасности (СЗЗ = 300 метров).

```
Workflow Claude:

1. cadastre_get_boundaries("50:30:0020101:50")
   → Границы промплощадки

2. geometry_buffer(промплощадка, distance_m=300)
   → Создание СЗЗ радиусом 300 м

3. infrastructure_find_nearby(lon, lat, radius=300, categories=["residential"])
   → Поиск жилых домов в СЗЗ

4. geometry_check_intersection(szz_buffer, жилые_районы)
   → Проверка пересечения СЗЗ с жилой застройкой

5. Результат:
   Обнаружено 3 жилых дома в СЗЗ
   Площадь пересечения: 4500 м²
   Статус: ❌ Нарушение СанПиН, требуется изменение расположения или сокращение производства

6. visualization_generate_static_map(
     center_coords,
     polygons=[промплощадка, СЗЗ, жилые_дома],
     markers=[конфликтные_точки]
   )
   → Карта для экспертизы СЭС (санитарно-эпидемиологической службы)
```

### Применение
- Проверка соблюдения СанПиН при размещении промышленных объектов
- Выявление конфликтов с жилой застройкой на ранней стадии
- Документирование для государственной экспертизы

## 4. Противопожарные разрывы (СП 18.13330.2019)

### Нормативная база
**СП 18.13330.2019** "Производственные объекты":
- Минимальные расстояния между зданиями зависят от степени огнестойкости и класса конструктивной пожарной опасности
- Минимальные расстояния между внешними сетями и зданиями

### Сценарий: Проверка проекта застройки

```
1. cadastre_get_boundaries("участок_А")
2. cadastre_get_boundaries("участок_Б")

3. geometry_measure_distance(здание_А_boundary, здание_Б_boundary)
   → Расстояние между зданиями: 8.5 м

4. Проверка по СП 18.13330.2019:
   Если здание А - класс C0 (непожароопасное)
   И здание Б - класс C1 (малопожароопасное)
   → Минимальный разрыв: 6 м
   → Статус: ✅ Соблюдено (8.5 > 6)

5. geometry_measure_distance(здание_А, газопровод)
   → Расстояние до газопровода: 3.2 м

6. Проверка по СП 18.13330.2019:
   Минимальное расстояние от газопровода среднего давления до здания: 4 м
   → Статус: ❌ Нарушение, требуется перенос трассы
```

### Применение
- Автоматическая проверка противопожарных разрывов
- Валидация проектов перед экспертизой
- Оптимизация размещения зданий на участке

## 5. Красные линии и линии регулирования застройки

### Нормативная база
**Красная линия** — граница, отделяющая территорию улично-дорожной сети от остальной территории.
- От красных линий отсчитываются минимальные отступы для строительства (обычно 3-5 м для жилых зданий)

### Сценарий: Проверка соответствия ГПЗУ

```
1. Загрузка красной линии из ГПЗУ (ручной ввод координат или GeoJSON)

2. geometry_buffer(красная_линия, distance_m=3)
   → Зона запрета строительства (отступ 3 м от красной линии)

3. cadastre_get_boundaries(участок)
4. Загрузка контура здания из проекта

5. geometry_check_intersection(здание, зона_запрета)
   → Проверка, что здание не пересекает зону запрета

6. Результат:
   Обнаружено пересечение: угол здания заходит на 1.2 м в зону отступа
   → Статус: ❌ Нарушение ГПЗУ, требуется корректировка проекта
```

### Применение
- Проверка проектов на соответствие ГПЗУ
- Автоматическая валидация перед подачей на экспертизу
- Выявление нарушений на ранней стадии проектирования

## 6. Инженерные сети и точки подключения

### Нормативная база
**Технические условия (ТУ)** ресурсоснабжающих организаций:
- Определяют место подключения
- Подключаемую нагрузку
- Условия подключения

**Внеплощадочные наружные сети** — инженерные сети от точек подключения (по ТУ) до границ земельного участка.

### Сценарий: Расчёт длин внеплощадочных сетей для сметы

```
1. cadastre_get_boundaries(cadastral_number)
   → Границы участка

2. Точки подключения (из ТУ):
   geocode_address_to_coords("Водопроводная станция №5")
   → lon: 37.617, lat: 55.753

   geocode_address_to_coords("ТП-123 Трансформаторная подстанция")
   → lon: 37.620, lat: 55.755

3. Расчёт расстояний до точек подключения:
   geometry_measure_distance(
     site_entry_point,
     water_station_coords
   )
   → Длина внеплощадочного водопровода: 450 м

   geometry_measure_distance(
     site_entry_point,
     transformer_coords
   )
   → Длина внеплощадочной ЛЭП: 320 м

4. Повторить для всех коммуникаций:
   - Водоснабжение (СП 31.13330)
   - Канализация (СП 32.13330)
   - Электроснабжение
   - Газоснабжение

5. infrastructure_calculate_distances(
     from_coords=[участок],
     to_addresses=[все_точки_подключения]
   )
   → Матрица расстояний для сметной документации
```

### Применение
- Автоматический расчёт длин внеплощадочных сетей
- Оценка стоимости подключения к инженерным сетям
- Документирование для ТУ и сметы

- Несовпадении кадастровых границ с фактическими
- Наложении границ соседних участков
- Самозахвате территории

### Сценарий: Судебная экспертиза земельного спора

```
"Проверь, пересекаются ли границы двух соседних участков."

1. cadastre_get_boundaries("50:21:0010401:14")
   → Участок А (собственник Иванов)

2. cadastre_get_boundaries("50:21:0010401:15")
   → Участок Б (собственник Петров)

3. geometry_check_intersection(участок_А, участок_Б)

4. Расчёт спорной территории:
   geometry_calculate_area(зона_пересечения)
   → Площадь наложения: 35.8 м²

5. visualization_generate_static_map(
     center_coords,
     polygons=[
       {geojson: участок_А, color: "blue"},
       {geojson: участок_Б, color: "red"},
       {geojson: зона_пересечения, color: "yellow"}
     ]

   → Карта конфликта для судебной экспертизы

6. visualization_export_geojson([участок_А, участок_Б, спорная_зона])
   → Экспорт для QGIS и детального геодезического анализа
```

## 11. Интеграция с ГИС (QGIS, AutoCAD Civil 3D)

### Рабочий процесс

**Этап 1: Claude получает данные через MCP**
```
1. cadastre_get_boundaries(cadastral_number)
   → Границы участка (GeoJSON)

2. infrastructure_find_nearby(lon, lat, radius=1000)
   → Инфраструктура вокруг (GeoJSON точек)

3. visualization_export_geojson([границы, инфраструктура])
   → Экспорт в файл site_analysis.geojson
```

**Этап 2: Инженер загружает в QGIS**
```
1. Открыть QGIS
2. Слой → Добавить слой → Векторный слой → site_analysis.geojson
3. Наложить на топографическую карту (OSM или Росреестр)
4. Добавить цифровую модель рельефа (ЦМР)
5. Выполнить анализ:
   - Трассировка инженерных сетей
   - Расчёт объёмов земляных работ
   - Анализ санитарных зон
   - Совмещение с генпланом
```

**Этап 3: Интеграция с AutoCAD Civil 3D**
```
1. Импорт GeoJSON через плагин или конвертацию в DXF
2. Координирование точек участков по кадастровым границам
3. Создание поверхности (Surface) на основе ЦМР
4. Размещение зданий с учётом красных линий и отступов
5. Проектирование инженерных сетей:
   - Санитарно-техническая канализация
   - Ливневая канализация
   - Водоснабжение
   - Электроснабжение
```

**Этап 4: Российское ПО (Renga, Кредо-диалог)**
```
1. Renga (BIM) — проектирование зданий
2. Кредо-диалог — создание ЦМР и геологических моделей
3. NEOSYNTES — управление инженерными данными
4. Обмен через IFC (Industry Foundation Classes)
```

### Применение
- Профессиональная обработка геоданных в ГИС
- BIM-проектирование с учётом контекста участка
- Соответствие требованиям Постановления №331 (обязательное BIM с 2024-2025)

## 12. BIM и интеграция с геоданными

### Концепция интеграции BIM и ГИС

**Роль ГИС:**
- Передаёт информацию об окружении проектируемого объекта в BIM
- Данные: кадастровые границы, рельеф, инженерные сети, соседние здания

**Роль BIM:**
- Детальное проектирование зданий и сооружений
- После строительства точная BIM-модель переносится в ГИС для эксплуатации

### Workflow интеграции

```
1. Claude → MCP → Росреестр/Yandex:
   cadastre_get_boundaries(cadastral_number)
   infrastructure_find_nearby(...)
   → Контекст участка (границы, инфраструктура, рельеф)

2. Экспорт в ГИС:
   visualization_export_geojson(...)
   → site_context.geojson

3. ГИС (QGIS) → обработка:
   - Наложение на топографию
   - Анализ рельефа
   - Зоны ограничений (СЗЗ, охранные зоны)
   → Экспорт в IFC или DWG

4. BIM (Revit, Renga):
   - Импорт контекста из ГИС
   - Проектирование зданий с учётом контекста
   - Проверка коллизий с существующей инфраструктурой

5. После строительства:
   - Исполнительная BIM-модель
   - Экспорт в ГИС для эксплуатации
   - Управление жизненным циклом объекта
```

### Статистика (2024-2025)
- **С 1 июля 2024** — обязательный переход на BIM при проектировании
- **С 1 января 2025** — обязательное использование BIM при СМР
- **20% застройщиков** применяют BIM (около 40% строительной площади)
- **К середине 2025** ожидается увеличение до 50%

### Применение
- Контекстное BIM-проектирование (здание + окружение)
- Соответствие Постановлению №331 и №1431
- Lifecycle management (жизненный цикл от проектирования до эксплуатации)

### Вариант 1: Анализ для инвестора (предпокупочная проверка)

**Задача:** Инвестор хочет купить участок для жилого комплекса. Нужна быстрая оценка потенциала.

```
"Проанализируй инвестиционный потенциал участка 77:01:0005001:1234."

Workflow Claude:

1. cadastre_get_info(cadastral_number)
   → Площадь: 1.5 га, ВРИ: многоэтажная жилая застройка, стоимость: 120 млн ₽

2. geometry_calculate_area(boundaries)
   → Проверка площади: 15120 м² ≈ 1.5 га ✅

3. Расчёт максимальных параметров застройки:
   Если ВРИ = "многоэтажная жилая застройка" → КП ≤ 1.2 (СП 42.13330.2016)
   Макс. общая площадь этажей = 15120 × 1.2 = 18144 м²

4. infrastructure_find_nearby(lon, lat, radius=500, categories=["school", "hospital", "metro"])
   → Школа: 320 м ✅
   → Больница: 680 м ⚠️ (за пределами шаговой доступности)
   → Метро: 850 м ⚠️

5. infrastructure_calculate_distances(from_coords, to_addresses=["Центр города", "ТТК"])
   → До центра: 12 км
   → До ТТК: 3 км

6. visualization_generate_static_map(...)
   → Карта для презентации инвесторам

Резюме для инвестора:
- Макс. жилая площадь: ~18000 м²
- При средней цене 150 тыс ₽/м² → потенциальная выручка: 2.7 млрд ₽
- Стоимость участка: 120 млн ₽ (4.4% от выручки) ✅ Хорошее соотношение
- Инфраструктура: школа в шаговой доступности, метро далековато
- Рекомендация: ✅ Участок перспективен для жилого комплекса комфорт-класса
```

**Результат:** За 3-5 минут — инвестиционный анализ с оценкой потенциала, рисков, максимальных параметров застройки.

### Вариант 3: Градостроительная экспертиза (проверка ГПЗУ)

**Задача:** Проверить соответствие проекта ГПЗУ перед подачей на государственную экспертизу.

```
"Проверь соответствие проекта Градостроительному плану земельного участка."

1. cadastre_get_info(cadastral_number)
   → ВРИ: индивидуальное жилищное строительство

2. Проверка соответствия ВРИ планируемому назначению:
   Планируется: частный дом ✅
   (Если бы планировался многоквартирный дом → ❌ Нарушение)

3. Из ГПЗУ: минимальный отступ от границ = 3 м

4. geometry_buffer(site_boundary, distance_m=3)
   → Зона запрета строительства вдоль границ

5. Загрузка контура дома из проекта (GeoJSON или координаты)

6. geometry_check_intersection(дом, зона_запрета)
   → Пересечение обнаружено!
   → Северо-западный угол дома заходит на 1.5 м в зону отступа

7. Рекомендация:
   ❌ Проект не соответствует ГПЗУ
   → Необходимо сдвинуть дом на 1.5 м к югу или востоку
   → Или уменьшить габариты дома

8. visualization_generate_static_map(
     polygons=[участок, дом, зона_запрета, зона_нарушения]
   )
   → Карта для корректировки проекта
```

**Результат:** Автоматическая проверка на соответствие ГПЗУ, выявление нарушений до подачи на экспертизу.

- Расстояния до поставщиков и баз

## Конкурентные преимущества

### 1. Естественный язык вместо сложных ГИС
- Общение с Claude на русском языке
- Не нужно учиться работать в QGIS или AutoCAD
- Автоматическое построение workflow из цепочки инструментов

### 2. Автоматизация типовых расчётов
- Экономия времени (2-3 минуты вместо часов ручной работы)
- Исключение арифметических ошибок
- Соответствие нормативной базе

### 3. Соответствие российским нормативам
- СП 42.13330.2016 (градостроительство)
- СП 18.13330.2019 (производственные объекты)
- СанПиН 2.2.1/2.1.1.1200-03 (СЗЗ)
- ГрК РФ (градостроительный кодекс)

### 4. Бесплатные источники данных
- Росреестр (НСПД) — бесплатно
- Yandex Maps бесплатный тариф (25000 запросов/день)
- Подходит для нечастого использования

### 5. Точность геодезических расчётов
- Площади: ±0.01 м² (UTM проекция)
- Расстояния: ±1 метр (геодезические расчёты WGS84)
- Профессиональный уровень точности

### 6. Экспорт в профессиональные системы
- GeoJSON для QGIS
- DWG/DXF для AutoCAD
- IFC для BIM (Revit, Renga)
- JSON для CRM и других систем
