## 🔌 Последний шаг: Подключение к Claude Code

### Шаг 1: Откройте файл конфигурации

```bash
nano ~/.claude/mcp-servers.json
```

Если файл не существует, создайте его.

### Шаг 2: Добавьте конфигурацию

Скопируйте и вставьте (замените `<path-to>/construction-maps-mcp` на путь до вашей копии репозитория):

```json
{
  "construction-maps": {
    "command": "/usr/local/bin/python3.14",
    "args": [
      "-m",
      "construction_maps_mcp"
    ],
    "cwd": "<path-to>/construction-maps-mcp",
    "env": {},
    "description": "MCP server for construction land parcel analysis - Rosreestr + Yandex Maps"
  }
}
```

**Если у вас уже есть другие MCP серверы**, добавьте секцию `"construction-maps"` внутрь существующего объекта:

```json
{
  "existing-server": {
    ...
  },
  "construction-maps": {
    "command": "/usr/local/bin/python3.14",
    "args": ["-m", "construction_maps_mcp"],
    "cwd": "<path-to>/construction-maps-mcp",
    "env": {}
  }
}
```

### Шаг 3: Сохраните и закройте

- В nano: `Ctrl+O` (сохранить), `Enter`, `Ctrl+X` (выйти)

### Шаг 4: Перезапустите Claude Code

Полностью закройте и откройте Claude Code заново.

## 💡 Особенности

### Кеширование

Все запросы кешируются автоматически:

- **Кадастровые данные**: 30 дней
- **Геокодирование**: 7 дней
- **Инфраструктура**: 1 день

Повторные запросы выполняются **мгновенно** из кеша!

Кеш хранится в: `~/.construction_maps_mcp/cache.db`

### Rate Limiting

API имеют ограничения на количество запросов:

- **Yandex Maps**: 15 запросов/минуту
- **Росреестр**: 30 запросов/минуту

MCP сервер автоматически контролирует лимиты.

### Точность расчётов

Геометрические операции используют:

- **pyproj** - геодезические вычисления на эллипсоиде WGS84
- **Shapely** - точные операции с полигонами
- **UTM проекция** - для расчёта площадей в метрах

Точность площадей: **±0.01 м²**
Точность расстояний: **±1 метр**

## 🔧 Устранение проблем

### Сервер не появляется в Claude

**Проверьте конфигурацию:**
```bash
cat ~/.claude/mcp-servers.json
```

Должен быть раздел `"construction-maps"`.

**Проверьте логи Claude Code:**
```bash
tail -f ~/.claude/logs/mcp-*.log
```

### Ошибка "Module not found"

```bash
# Переустановите зависимости
cd ~/construction-maps-mcp
/usr/local/bin/python3.14 -m pip install --user -e .
```

### Ошибка "YANDEX_MAPS_API_KEY not set"

Проверьте `.env`:
```bash
cat .env | grep YANDEX_MAPS_API_KEY
```

Должен быть: `YANDEX_MAPS_API_KEY=YOUR_YANDEX_KEY`

### Медленная работа

- Первый запрос к API всегда медленный (реальное обращение к Росреестру/Yandex)
- Повторные запросы **быстрые** (из кеша)
- Проверьте скорость интернета

### Debug режим

Включите подробные логи:

```bash
# Отредактируйте .env
nano .env

# Измените
LOG_LEVEL=DEBUG
LOG_FORMAT=console
```

Перезапустите Claude Code.
