**Изменение 2: Исправлено получение метаданных (строки 133-141, 195-203)**

Проблема: Библиотека rosreestr2coord не предоставляет атрибуты `area`, `address`, `category` на объекте Area.

Решение: Убраны попытки получения несуществующих атрибутов, добавлены значения по умолчанию с пояснением.

```python
# Было:
result["area_m2"] = await asyncio.to_thread(getattr, area, "area", 0)
result["address"] = await asyncio.to_thread(getattr, area, "address", "Адрес не указан")
result["category"] = await asyncio.to_thread(getattr, area, "category", "Категория не указана")

# Стало:
# Note: rosreestr2coord doesn't provide metadata (area, address, category)
# These would need to be fetched separately from Rosreestr API
result["area_m2"] = 0
result["address"] = "Адрес не указан (требуется доп. запрос)"
result["category"] = "Категория не указана (требуется доп. запрос)"
```

**Примечание**: Для получения полных метаданных требуется дополнительный запрос к публичному API Росреестра или парсинг HTML-страниц.

## ⚠️ Известные ограничения

### 1. Метаданные из Росреестра
**Проблема**: Библиотека rosreestr2coord возвращает только геометрию, без метаданных (площадь, адрес, категория).

**Решение**: Добавлены значения по умолчанию с пояснением. Для получения полных метаданных потребуется:
- Дополнительный запрос к публичному API Росреестра
- Или парсинг HTML-страниц с сайта Росреестра

**Статус**: Not a bug, known limitation

### 2. Слабый валидатор кадастровых номеров
**Проблема**: Валидатор пропускает невалидные номера вроде "invalid".

**Решение**: Отмечено как known issue в тесте.

**Статус**: Low priority, не блокирует работу
